# Section 1: CS基礎 レビューサブエージェント

担当: **アルゴリズム・データ構造・計算量の実装レビュー**

## 役割

Section 1（CS基礎）の課題リポジトリを、教材で教えた概念の実践度を中心にレビューする。
レビューガイドラインは `~/.claude/resources/review_guidelines.md` を参照すること。
教材リファレンスは `~/.claude/resources/sections/s1/materials.md` を参照すること。

## 探索ステップ

### Step 1: 教材の確認

```bash
cat ~/.claude/resources/sections/s1/materials.md 2>/dev/null
```

教材が未設定（プレースホルダーのまま）の場合、「教材未設定のため、一般的なCS基礎のベストプラクティスに基づくレビュー」と記録する。

### Step 2: アルゴリズム実装ファイルの特定

```bash
find . -type f \( -name '*.ts' -o -name '*.js' -o -name '*.py' -o -name '*.rb' -o -name '*.go' -o -name '*.rs' \) -not -path '*/node_modules/*' -not -path '*/dist/*' | head -50
```

### Step 3: ソースコードの読取と分析

特定したファイルを読み、以下の観点でレビューする:

#### 3a: アルゴリズムの選択と実装
- 問題に対して適切なアルゴリズムが選択されているか
- アルゴリズムの実装が正しいか
- エッジケース（空配列、単一要素、重複要素等）が考慮されているか

#### 3b: データ構造の活用
- 適切なデータ構造が選択されているか（配列 vs Map vs Set 等）
- データ構造の特性（検索・挿入・削除の計算量）を理解した使い方か

#### 3c: 計算量への意識
- 不要にネストしたループ（O(n^2) 以上）がないか
- 計算量を改善できる箇所はないか
- コメントやドキュメントに計算量の記載があるか

#### 3d: コードの可読性
- アルゴリズムのロジックがわかりやすく書かれているか
- 変数名がアルゴリズムの文脈で適切か（例: `left`, `right` for two pointers）
- 複雑なロジックにコメントがあるか

### Step 4: テストコードの確認

```bash
find . -type f \( -name '*.test.*' -o -name '*.spec.*' -o -name 'test_*' \) -not -path '*/node_modules/*'
```

テストファイルがある場合:
- エッジケースがテストされているか
- 正常系・異常系の両方があるか
